### 🧠  How I Think About Systems

```text
                     ┌──────────────────────┐
                     │   Business Domain    │
                     └──────────┬───────────┘
                                │
                     Contract-First Design
                                │
             ┌──────────────────▼──────────────────┐
             │        Domain-Driven Services       │
             │      Bounded Contexts · SOLID       │
             └───────────┬─────────────┬───────────┘
                         │             │
                    REST / JWT      Events / Kafka
                         │             │
              ┌──────────▼───┐   ┌────▼────────────┐
              │ Synchronous  │   │  Asynchronous   │
              │    APIs      │   │  Event Flows    │
              └──────────┬───┘   └────┬────────────┘
                         │             │
                    ┌────▼─────────────▼────┐
                    │   Distributed Data    │
                    │ Aurora · PostgreSQL   │
                    │ Redis · Caffeine      │
                    └──────────┬────────────┘
                               │
                    ┌──────────▼────────────┐
                    │ Cloud-Native Runtime  │
                    │ AWS EKS · Kubernetes  │
                    │ ArgoCD · CI/CD        │
                    └──────────┬────────────┘
                               │
                    ┌──────────▼────────────┐
                    │     Observability     │
                    │ DataDog · Prometheus  │
                    │ Grafana · Elastic     │
                    └───────────────────────┘
```

> I design distributed systems from the failure modes backward: consistency, idempotency, observability, recovery, and scale are architecture concerns — not production afterthoughts.
